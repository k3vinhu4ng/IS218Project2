AAA Testing

Arrange - Act - Assert

Introduction

AAA Testing refers to Arrange, Act, and Assert, which is a pattern for writing good tests for our programs. Testing helps us know if our program works or if our program is full of errors.

The AAA Pattern

The AAA pattern is great for testing.

Arrange

This step arranges the inputs and targets. This involves handling any operations needed for the test. This can include objects, special settings, prepping a database, logging into a web app, etc

Act

This refers to acting on the target behavior. The Act step covers the main thing to be test. This can involve calling a method or function, calling a REST API, or interacting with a webpage.

Assert

Assert is the last step, and it refers to asserting the expected outcomes. The actions done in the Act step should produce some type of response. Assert aims to verify the response. In simple cases, Assert can mean checking numeric or string values. In more complex cases, Assert may be checking multiple facets of a system. Assert basically determines whether or not the test passes or fails.

Unit Testing

Let us take a look at how AAA is used in Python Unit Testing.

We will be writing a Unit Test using AAA for the function below.

def addition(num1 + num2):
    return num1 + num2

Now we will write a Unit Test for the function addition.

def test_addition():
    #Arrange
    value1 = 5
    value2 = 3
   
    #Act
    answer = addition(value1, value2)
   
    #Assert
    Assert answer == 8

For the Arrange step, we set test values for two variables that will be put into our function. Then, we Acted on our function by calling it. Lastly, we Asserted that our answer is 8, since our initial test values were 5 and 3.
